# DreamStudio 2.0 改造方案

> 核心理念：不是三个独立的"房间"，而是同一个创作空间的**不同放大倍数**。
> 用户始终在同一个项目里，根据需求选择用「说」「看」或「调」来创作，界面随意图渐进展开，零门槛，无割裂。

---

## 一、整体信息架构

```
DreamStudio 2.0
│
├── 🏠 首页（单一入口）
│   └── 输入框："描述你想创作的内容…" + 可选上传文档/图片
│
└── 🎬 项目空间（统一画布，三个可切换的"聚焦层"）
    │
    ├── 💬 对话层 —— 说想法，快速迭代
    ├── 🎬 故事板层 —— 看全局，编排序列
    ├── 🔧 节点层 —— 调细节，自定义流程
    │
    └── 🎨 资产抽屉（全局侧边栏，随时呼出）
```

**关键设计原则：**
- 无模式切换：三层共用一套数据模型，任何层的修改实时同步到其他层
- 向下钻取：任意层遇到"🔧"标记的元素，可展开更细粒度的控制面板
- 状态同步：全局只有一个"当前焦点实体"（镜头/节点/消息），三层都响应焦点变化

## 二、现有架构 vs 目标架构

### 现有架构（两态切换）

```
HomePage ←→ Planning/Workspace
                 ↓
    ┌────────────┴────────────┐
    ↓                         ↓
规划态                      执行态
├── 左侧: Console          ├── 左侧: Console
├── 中间: PlanPreview      ├── 中间: NodeCanvas
└── 右侧: ContextPanel     └── 右侧: NodeWorkspace
```

### 目标架构（三层穿透）

```
HomePage → 项目空间（统一画布，三层切换）
              ↓
    ┌─────────┴─────────┐
    ↓         ↓         ↓
对话层    故事板层    节点层
    ↓         ↓         ↓
    └─────────┴─────────┘
              ↓
      资产抽屉（全层共用）
```

### 3.6 💬 对话层右侧预览区（自适应预览面板）

#### 核心定位
> 右侧预览区 = **故事板的微缩版**（mini storyboard），不是独立的资产库。
> 故事板是"序列化的结果集"，对话右侧是"结果预览窗口"。

#### 两种模式

| 模式 | 定义 | 典型内容 | 右侧预览区展示 |
|------|------|----------|----------------|
| **资产模式** | 没有时序关系，内容独立存在 | 单张图片、角色卡合集、场景参考图、剧本文档 | 资产网格/详情视图 |
| **序列模式** | 内容按时间或逻辑顺序排列 | 视频分镜、漫画格子、幻灯片页面 | 故事板序列视图 |

用户可以在两种模式之间**无缝转换**：从资产模式开始，当用户开始排列顺序（例如"把这些角色卡按出场顺序排好"），系统自动切换到序列模式。

## 四、钻取面板：场景化"查看改了什么"

### 核心原则
> 用户点击"查看改了什么"时，看到的是**本次修改的语义化解释**，而不是固定格式的技术面板。根据修改类型，展示最相关的信息层级。

### 4.1 通用交互规范

1. **触发方式**：点击"查看改了什么 🔧"或修改标记（⚡）
2. **展示位置**：当前视图的**右侧滑出抽屉**（宽度360px）或**居中浮层**（根据内容量）
3. **关闭方式**：点击外部、按ESC、或点击关闭按钮
4. **内容结构**：
   - 标题：修改类型 + 影响范围
   - 对比区：根据场景（文本/资产/参数/视觉/列表）
   - 操作区：1~3个主要操作
5. **动画**：从右侧滑入，伴随背景半透明遮罩

## 十、关键技术点

1. **虚拟滚动**：故事板使用 `react-window` 或类似库支持 100+ 镜头
2. **Framer Motion**：视图切换动画、DrillPanel 滑入动画
3. **Floating UI**：下拉/弹层自动避让视口边缘
4. **Zustand 状态同步**：三层共用 store，通过 `focusedEntity` 同步焦点
5. **修改记录**：后端每次修改生成 Modification 记录，前端按 type 渲染

---

## 十一、注意事项

1. **版本 ID 混淆**：`v.id` = 数据库主键，`v.versionNo` = 用户版本号
2. **props 层层传递**：NodeWorkspace 的 props 通过 tabContentMap 传递，修改时注意检查传递链路
3. **CSS 样式冲突**：大量使用 CSS 类名，优先使用组件级别 CSS 文件
4. **API 错误处理**：异步操作要 try-catch，注意检查 nodeId 是否变化
5. **钻取面板关闭**：点击外部、按ESC、或点击关闭按钮，需同步清理状态
