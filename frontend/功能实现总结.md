## 三、数据结构约定

### 节点数据 (node.data)
```typescript
{
  // 基础字段
  result?: string;           // 结果内容
  status?: 'idle' | 'running' | 'completed' | 'failed' | 'stale';
  isLocked?: boolean;

  // 版本相关
  currentVersion?: number;
  baseVersion?: number;
  isRevised?: boolean;
  versionHistory?: Version[];

  // 提案相关
  proposal?: Proposal;
  proposalStatus?: 'pending' | 'confirmed' | 'rejected';

  // 节点类型特定
  characters?: Character[];
  scenes?: Scene[];
  prompts?: Prompt[];
  videos?: Video[];
  // ...
}
```

### 提案结构 (Proposal)
```typescript
{
  id: string;
  type: 'add_node' | 'remove_node' | 'update_config' | 'revision';
  title: string;
  description: string;
  changes: Change[];
  status: 'pending' | 'applied' | 'rejected';
}
```

### 版本结构 (Version)
```typescript
{
  version: string;        // 'v3-r2'
  type: '运行版' | '修订版';
  time: string;          // '10分钟前'
  isCurrent: boolean;
  data?: object;         // 完整的节点数据
}
```

## 四、前端状态管理

### Store 架构
```
useUIStore
├── currentView: 'home' | 'planning' | 'workspace'
├── selectedNodeId: string
├── isCanvasFullscreen: boolean
└── 布局状态

useWorkflowStore
├── nodes: Node[]
├── connections: Connection[]
├── viewport: {x, y, zoom}
└── 方法: updateNodeData, markDownstreamAsStale, etc.

useProjectStore
├── currentProjectId: string
├── currentVersion: Version
├── versions: Version[]
└── 保存状态
```

### 关键派生状态 (App.jsx useMemo)
```
selectedNode = canvasNodes.find(n => n.id === selectedNodeId)
downstreamNodes = 根据selectedNode和connections计算
upstreamNodes = 根据selectedNode和connections计算
runButtonText = 根据stale状态动态切换
```
