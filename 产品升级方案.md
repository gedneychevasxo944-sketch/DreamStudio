# DreamStudio 2.0 优化方案 —— 分层穿透式创作空间

> 核心理念：不是三个独立的“房间”，而是同一个创作空间的**不同放大倍数**。  
> 用户始终在同一个项目里，根据需求选择用「说」「看」或「调」来创作，界面随意图渐进展开，零门槛，无割裂。

---

## 一、整体信息架构

```
DreamStudio 2.0
│
├── 🏠 首页（单一入口）
│   └── 输入框：“描述你想创作的内容…” + 可选上传文档/图片
│
└── 🎬 项目空间（统一画布，三个可切换的“聚焦层”）
    │
    ├── 💬 对话层 —— 说想法，快速迭代
    ├── 🎬 故事板层 —— 看全局，编排序列
    ├── 🔧 节点层 —— 调细节，自定义流程
    │
    └── 🎨 资产抽屉（全局侧边栏，随时呼出）
```

**关键设计原则：**
- 无模式切换：三层共用一套数据模型，任何层的修改实时同步到其他层
- 向下钻取：任意层遇到“🔧”标记的元素，可展开更细粒度的控制面板
- 状态同步：全局只有一个“当前焦点实体”（镜头/节点/消息），三层都响应焦点变化

## 二、三层统一界面（优化后的视觉与交互）

### 2.1 顶部导航栏（三层共用）

```
┌─────────────────────────────────────────────────────────────────┐
│ ← 项目名称（可编辑）          [💬 对话] [🎬 故事板] [🔧 节点]    ⋮ 菜单 │
│                                                       👤 头像     │
│ 当前焦点：镜头3 · 场景美术Agent（点击可切换焦点）        🔍 缩放   │
└─────────────────────────────────────────────────────────────────┘
```

- **视角切换**：点击标签切换聚焦层，带有**平滑过渡动画**（对话气泡“缩小”嵌入故事板左侧，或节点图“展开”为全画布）
- **焦点指示器**：始终显示当前选中的实体（镜头编号/节点名称），并支持下拉快速跳转
- **缩放控制**：仅在节点层有效，其他层灰显

**优化点：**
- 每个系统回复都附带 **“查看改了什么 🔧”** ，点击后展开**Prompt对比面板**（轻量钻取）
- 对话中可**直接呼出资产库**（通过输入框下方的按钮）
- 连续3次调整同一镜头但未点“满意”时，底部出现**智能提示条**（非模态）：“检测到多次调整，是否切换到故事板批量处理？” [稍后] [打开故事板]

**优化点：**
- 故事板采用**虚拟滚动**，支持100+镜头流畅浏览
- 每个镜头的缩略图右下角显示**修改标记**（⚡），悬停显示修改来源（如“来自对话调整”）
- **拖拽资产库的角色卡到镜头**：镜头边框高亮，提示“替换角色”或“添加为前景”，松手后自动关联一致性ID
- 右侧预览面板支持**直接钻取**：点击“🔧 编辑参数”后，在当前故事板内弹出**参数面板浮层**（无需切换视图），可快速修改Prompt/模型等，修改后实时更新缩略图

**优化点：**
- **默认“简化视图”**：只显示主流程节点，中间处理节点收拢成“⋯”，用户可手动“全部展开”
- **节点分组**：相关节点（如角色美术、背景美术）自动成组，组标题可折叠，组内节点以迷你卡片显示
- **焦点高亮**：当前编辑节点上下游高亮，其他节点半透明，减少认知负担
- **锁定机制可视化**：若节点被下游锁定，卡片上显示 🔒 并带工具提示：“被节点‘合成节点’锁定，请先解锁下游”
- **运行反馈**：工作流执行时，节点边框有进度光效；完成后全局Toast通知，并汇总错误（若有）到顶部栏

- 资产可标记“常用”，自动出现在每个项目的推荐区

## 三、关键交互机制（细化）

### 3.1 “向下钻取”规范

- **可钻取元素**：对话中的“查看改了什么 🔧”、故事板中的“🔧 编辑参数”、节点卡片的任何配置项
- **钻取方式**：单击 🔧 或带图标按钮 → 在当前界面弹出**轻量参数面板**（浮层或侧边抽屉），**不跳转视图**
- **面板内容**：根据上下文，显示Prompt对比、参数滑块、模型选择等。面板底部有“在节点层完整编辑”链接，点击后切换到节点层并定位
- **关闭钻取**：点击面板外区域或按ESC关闭。若用户做了修改，关闭前提示保存

### 3.2 全局选中模型

- 项目中只有一个**当前焦点实体**，类型为：`Message | Shot | Node`
- 焦点变化时，所有层同步高亮对应实体（若存在映射）
  - 对话层选中一条消息 → 故事板自动滚动到该消息关联的镜头（若有关联）
  - 故事板选中一个镜头 → 节点层高亮对应的“场景美术Agent”节点
  - 节点层选中一个节点 → 故事板高亮对应的镜头（若无对应，顶部栏显示“当前节点不指向特定镜头”）
- 焦点切换的动画：**平滑滚动 + 短暂呼吸光效**

### 3.3 智能建议系统（防打扰）

- **展示形式**：仅在界面右下角以**非模态通知条**出现，带关闭按钮
- **触发规则**：
  - 连续3次对话调整同一镜头且未点“满意” → 建议切换到故事板
  - 用户在故事板反复修改同一个参数 → 建议保存为预设
  - 用户在节点层频繁编辑但未运行 → 建议“试运行当前节点”
- **用户控制**：每个建议类型首次出现时，附带“不再显示此类建议”选项

### 3.4 修改溯源与版本管理

- 所有修改记录统一存储在项目历史中
- 在故事板镜头卡片上，修改标记（⚡）悬停显示：
  ```
  该镜头被修改过
  来源：对话调整（2分钟前）→ “背景太乱，要简洁一点”
  Prompt变化：12处
  [查看详情] [回退到此版本]
  ```
- 版本列表（顶部菜单“版本历史”）支持：
  - 自动生成版本摘要（如“镜头3背景简化”）
  - 对比两个版本的Prompt差异（高亮显示）
  - 删除版本时提示影响范围（如“下游有2个节点引用此版本”）

### 3.5 节点锁定机制（透明化）

- 节点被锁定时，卡片右上角显示 🔒，鼠标悬停显示具体锁定源：“被节点‘合成节点’锁定”
- 在ConfigTab中，被锁字段灰显并带工具提示：“此字段被‘合成节点’锁定，请先解锁”
- 提供**锁定传播可视化**：点击节点上的锁图标，高亮显示从该节点到下游锁定节点的连线

### 3.6 空状态设计（全覆盖）

| 位置 | 无数据时显示 |
|------|-------------|
| 首页“最近项目” | 插画 + “暂无项目，点击下方输入框开始创作” + 示例提示 |
| 故事板（无镜头） | “还没有镜头，让AI帮你生成吧” + [💬 去对话生成] |
| 资产库（空） | “上传你的第一张角色卡或场景图” + [上传] |
| 版本历史（空） | “保存版本后，这里会显示历史记录” |
| ImpactTab（空） | “暂无依赖影响记录” |

## 四、用户旅程示例（优化版）

### 新手用户A（只想快速尝鲜）
1. 首页输入：“做一只赛博朋克猫”
2. AI自动生成10个镜头，展示在对话层
3. 用户看完，点击“下载” → 完成（全程未切换视图，< 5分钟）

### 进阶用户B（有想法，愿意微调）
1. 上传剧本 → 对话层生成分镜
2. 不满意第3镜背景 → 对话调整：“背景简洁点”
3. 系统调整后，用户点击“查看改了什么 🔧”，弹出Prompt对比，微调后关闭
4. 切换到故事板，拖拽资产库的角色卡到第3镜，自动替换角色
5. 批量导出 → 完成（30分钟，使用对话+故事板+轻量钻取）

### 技术用户C（搭建模板）
1. 新建项目，直接进入节点层
2. 从空白搭建工作流（拖拽节点、连线）
3. 保存为“我的赛博朋克模板”
4. 下次从模板创建，批量运行 → 资产库积累多个角色

## 五、技术实现要点（简述）

- **数据层**：统一状态管理（如Zustand），存储`shots`、`nodes`、`messages`及三者映射关系
- **渲染层**：React/Vue + 画布库（如React Flow用于节点层），虚拟滚动列表用于故事板
- **动画**：Framer Motion统一动效，视角切换使用`layoutId`共享元素动画
- **性能**：节点层默认简化视图（只渲染可见区域节点），资产库图片懒加载
- **边界检测**：下拉/弹层使用`floating-ui`自动避让视口边缘
