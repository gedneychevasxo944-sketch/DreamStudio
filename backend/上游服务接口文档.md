# 上游AI服务接口文档

> 本文档用于向上游服务提需求，定义 DreamStudio 平台需要的 AI 能力接口。当前所有接口均为本地模拟实现，需要替换为真实的上游服务调用。

---

## 接口概览

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取Agent列表 | GET | `/api/agents` | 获取所有AI Agent定义 |
| 获取Agent详情 | GET | `/api/agents/{id}` | 根据ID获取单个Agent |
| 发送聊天消息 | POST | `/api/chat` | 与AI Agent对话 |
| 获取聊天历史 | GET | `/api/chat/history` | 获取项目对话历史 |
| 执行工作流 | POST | `/api/execute` | 执行DAG工作流，SSE流式返回 |

---

## 1. 获取Agent列表

获取系统中所有可用的AI Agent（节点类型）定义，包括各Agent的输入输出端口配置。

### 请求

```
GET /api/agents
```

### 响应

```json
{
  "agents": [
    {
      "id": 1,
      "type": "producer",
      "name": "资深影视制片人",
      "category": "官方认证",
      "icon": "Target",
      "color": "#3b82f6",
      "description": "资深影视制片人，负责项目立项和整体规划",
      "inputs": [
        {
          "id": "idea",
          "label": "原始想法",
          "type": "text"
        }
      ],
      "outputs": [
        {
          "id": "proposal",
          "label": "项目立项书",
          "type": "document"
        }
      ],
      "createdAt": "2024-01-01T00:00:00"
    }
  ],
  "total": 7
}
```

### 响应字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| agents | Array | 是 | Agent列表 |
| agents[].id | Long | 是 | Agent唯一标识 |
| agents[].type | String | 是 | Agent类型代码，如 producer, content, visual |
| agents[].name | String | 是 | Agent显示名称 |
| agents[].category | String | 否 | 分类，如"官方认证" |
| agents[].icon | String | 否 | 图标名称，用于前端渲染 |
| agents[].color | String | 否 | 主题色，hex格式 |
| agents[].description | String | 否 | Agent描述 |
| agents[].inputs | Array | 是 | 输入端口定义 |
| agents[].outputs | Array | 是 | 输出端口定义 |
| agents[].inputs[].id | String | 是 | 端口ID |
| agents[].inputs[].label | String | 是 | 端口显示名 |
| agents[].inputs[].type | String | 是 | 数据类型：text, document, image-prompt, video, prompt-package, any |
| total | Integer | 是 | Agent总数 |

### 当前定义的Agent类型

| type | name | 作用 |
|------|------|------|
| producer | 资深影视制片人 | 分析项目可行性，生成项目立项书 |
| content | 金牌编剧 | 根据立项书编写分场剧本 |
| visual | 概念美术总监 | 分析剧本，生成角色/场景/道具设计及文生图指令 |
| director | 分镜导演 | 根据剧本和美术设计生成分镜脚本 |
| technical | 视频提示词工程师 | 优化视频生成提示词，配置生成参数 |
| videoGen | 视频生成 | 调用视频生成模型，产出视频 |
| videoEditor | 视频剪辑 | 剪辑、合成最终成片 |

## 3. 发送聊天消息

向指定的AI Agent发送消息并获取响应。支持多种返回类型（文本、图片、视频、Markdown、工作流）。

### 请求

```
POST /api/chat
Content-Type: application/json
```

### 请求体

```json
{
  "projectId": 123,
  "projectVersion": 1,
  "agentId": "producer",
  "agentName": "资深影视制片人",
  "message": "我想做一个关于都市年轻人追梦的微电影",
  "chatType": "agent"
}
```

### 请求字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| projectId | Long | 是 | 项目ID |
| projectVersion | Integer | 是 | 项目版本号 |
| agentId | String | 是 | Agent类型代码 |
| agentName | String | 否 | Agent显示名 |
| message | String | 是 | 用户发送的消息内容 |
| chatType | String | 否 | 聊天类型，默认"agent" |

### 响应

```json
{
  "id": 1704067200000,
  "projectId": 123,
  "projectVersion": 1,
  "agentId": "producer",
  "agentName": "资深影视制片人",
  "question": "我想做一个关于都市年轻人追梦的微电影",
  "thinkingSteps": [
    "正在分析项目可行性...",
    "评估预算和资源需求...",
    "制定项目时间线...",
    "生成项目立项书..."
  ],
  "resultType": "document",
  "result": "项目立项完成\n\n项目名称：都市情感微电影\n预算：500万\n周期：6个月\n\n项目概述：\n一部探讨现代都市人情感生活的微电影...",
  "workflowCreated": false,
  "workflowNodes": null,
  "workflowEdges": null,
  "createTime": "2024-01-01T12:00:00"
}
```

### 响应字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | Long | 是 | 消息唯一ID |
| projectId | Long | 是 | 项目ID |
| projectVersion | Integer | 是 | 项目版本号 |
| agentId | String | 是 | Agent类型代码 |
| agentName | String | 是 | Agent显示名 |
| question | String | 是 | 用户原始问题 |
| thinkingSteps | Array | 是 | AI思考步骤（用于前端展示推理过程） |
| resultType | String | 是 | 结果类型：text, document, image, video, markdown, workflow |
| result | String | 是 | 主要结果内容 |
| workflowCreated | Boolean | 否 | 是否创建了工作流（仅agentId为"assistant"时可能为true） |
| workflowNodes | Array | 否 | 工作流节点列表（创建工作流时返回） |
| workflowEdges | Array | 否 | 工作流边列表（创建工作流时返回） |
| createTime | String | 是 | 创建时间，ISO格式 |

### resultType 对应的数据结构

| resultType | result 格式 | 说明 |
|------------|-------------|------|
| text | 纯文本 | 普通文本回复 |
| document | 纯文本 | 文档内容（项目立项书、剧本等） |
| markdown | Markdown文本 | 格式化文档 |
| image | URL列表，逗号分隔 | 图片URL列表 |
| video | 单个URL | 视频URL |
| workflow | 工作流JSON | 自动创建的工作流定义 |

### 特殊处理：智能助理（assistant）

当 `agentId` 为 `"assistant"` 时，支持以下特殊功能：

1. **工作流创建**（message包含"创意"）
   - 返回 `workflowCreated: true`
   - 返回预定义的工作流节点和边

2. **图片生成**（message包含"图片"或"image"）
   - 返回图片URL列表

3. **视频生成**（message包含"视频"或"video"）
   - 返回视频URL

4. **Markdown格式**（message包含"markdown"或"格式"）
   - 返回Markdown格式文档

## 5. 执行工作流

执行指定的DAG工作流，通过SSE（Server-Sent Events）流式返回执行过程。

### 请求

```
POST /api/execute
Content-Type: application/json
```

### 请求体

```json
{
  "executionId": 1,
  "projectId": 123,
  "projectVersion": 1,
  "dag": {
    "nodes": [
      {
        "id": "producer-1",
        "type": "producer",
        "config": {}
      },
      {
        "id": "content-1",
        "type": "content",
        "config": {}
      }
    ],
    "edges": [
      {
        "from": "producer-1",
        "to": "content-1"
      }
    ]
  }
}
```

### 请求字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| executionId | Long | 否 | 执行记录ID（用于断点续传） |
| projectId | Long | 是 | 项目ID |
| projectVersion | Integer | 是 | 项目版本号 |
| dag | Object | 是 | 有向无环图（DAG）定义 |
| dag.nodes | Array | 是 | 节点列表 |
| dag.nodes[].id | String | 是 | 节点唯一ID |
| dag.nodes[].type | String | 是 | 节点类型（对应Agent类型） |
| dag.nodes[].config | Object | 否 | 节点配置参数 |
| dag.edges | Array | 是 | 边列表（节点间的数据流向） |
| dag.edges[].from | String | 是 | 源节点ID |
| dag.edges[].to | String | 是 | 目标节点ID |

### SSE事件流

执行过程会推送多种类型的SSE事件，客户端通过 `event: message` 接收。

#### 5.1 初始化事件 (init)

工作流开始执行时推送，包含已完成节点的信息（用于断点续传）。

```json
{
  "type": "init",
  "executionId": 1,
  "completedNodes": [
    {
      "nodeId": "producer-1",
      "nodeType": "producer",
      "status": "completed",
      "thinking": ["分析项目可行性...", "评估预算..."],
      "result": "项目立项完成..."
    }
  ]
}
```

#### 5.2 状态事件 (status)

节点状态变更时推送。

```json
{"type": "status", "nodeId": "producer-1", "status": "running", "content": "[资深影视制片人] 开始执行"}
{"type": "status", "nodeId": "producer-1", "status": "completed", "content": "[资深影视制片人] 执行完成"}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| type | String | 固定为 "status" |
| nodeId | String | 节点ID |
| status | String | 状态：running, completed, failed |
| content | String | 状态描述 |

#### 5.3 思考事件 (thinking)

节点推理过程中的思考步骤。

```json
{"type": "thinking", "nodeId": "producer-1", "content": "[资深影视制片人] 正在分析项目可行性...", "index": 0}
{"type": "thinking", "nodeId": "producer-1", "content": "[资深影视制片人] 评估预算和资源需求...", "index": 1}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| type | String | 固定为 "thinking" |
| nodeId | String | 节点ID |
| content | String | 思考步骤内容 |
| index | Integer | 步骤序号（从0开始） |

#### 5.4 图片事件 (images)

节点生成的图片结果。

```json
{
  "type": "images",
  "nodeId": "visual-1",
  "images": [
    "https://example.com/image1.jpg",
    "https://example.com/image2.jpg"
  ]
}
```

#### 5.5 视频事件 (videos)

节点生成的视频结果。

```json
{
  "type": "videos",
  "nodeId": "videoGen-1",
  "videos": [
    {
      "url": "https://example.com/video1.mp4",
      "title": "城市日出空镜",
      "duration": 5,
      "description": "5秒城市日出场景"
    }
  ]
}
```

## 模拟数据与真实接口的替换说明

当前实现中，所有接口均返回本地模拟数据。要替换为真实调用，需要：

1. 修改 `UpstreamAiClient` 类中的各方法实现
2. 使用 HTTP Client（如 RestTemplate 或 WebClient）调用上游服务
3. 将上游返回的数据映射到现有的 DTO 结构

### 替换示例

```java
// 当前模拟实现
public List<AgentDTO.Response> getAgents() {
    log.info("Fetching agents from upstream service: {}", baseUrl);
    return getDefaultAgents();  // 返回本地模拟数据
}

// 替换为真实调用
public List<AgentDTO.Response> getAgents() {
    log.info("Fetching agents from upstream service: {}", baseUrl);
    String url = baseUrl + "/api/agents";
    ResponseEntity<List<AgentDTO.Response>> response = restTemplate.exchange(
        url,
        HttpMethod.GET,
        null,
        new ParameterizedTypeReference<List<AgentDTO.Response>>() {}
    );
    return response.getBody();
}
```
