### 问题描述
`api.js` 中的 `createSSEConnection` 使用原生 `EventSource`，但 EventSource 不支持自定义 headers，导致 JWT token 无法发送到后端。

```javascript
// api.js line 97-103
const eventSource = new EventSource(`${API_BASE_URL}${url}`, {
  headers,  // EventSource 忽略 headers
});
```

### 当前状态
已使用 `fetch` + 流式处理（`createSSEConnectionForExecution`）绕过此问题，用于 `/workspace/chat` 接口。

### 建议方案
**方案A（推荐）：** 保持现状
- SSE 连接已改用 fetch + 流式处理
- 不需要修改

**方案B：** 迁移到 WebSocket
- 使用 `socket.io` 或原生 WebSocket
- 需要后端配合改造
- 成本：中高

## 2. 前端无 ErrorBoundary

### 问题描述
`src/main.jsx` 中没有 ErrorBoundary 组件，任何未捕获的错误会直接崩溃整个应用。

```javascript
// main.jsx
createRoot(document.getElementById('root')).render(
  <StrictMode>
    <App />
  </StrictMode>,
)
```

### 影响
- 用户操作时出错会导致白屏
- 无法优雅降级
- 生产环境难以定位问题

### 建议方案

```jsx
// src/components/ErrorBoundary.jsx
class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false, error: null };
  }

  static getDerivedStateFromError(error) {
    return { hasError: true, error };
  }

  componentDidCatch(error, errorInfo) {
    console.error('ErrorBoundary caught:', error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      return (
        <div style={{ padding: 40, textAlign: 'center' }}>
          <h2>出错了</h2>
          <p>抱歉，发生了未知错误</p>
          <button onClick={() => window.location.reload()}>刷新页面</button>
        </div>
      );
    }
    return this.props.children;
  }
}
```

```jsx
// main.jsx
import ErrorBoundary from './components/ErrorBoundary';

createRoot(document.getElementById('root')).render(
  <StrictMode>
    <ErrorBoundary>
      <App />
    </ErrorBoundary>
  </StrictMode>,
)
```

**成本：** 低（1-2小时）
**风险：** 低

## 3. useEffect 依赖数组不完整

### 问题描述
部分 useEffect 使用了外部变量但依赖数组未包含，导致可能使用 stale 值。

**示例 - VisualNodeContent:**
```javascript
// line 670-680
useEffect(() => {
  if (characters.length > 0 || scenes.length > 0 || props.length > 0) {
    onUpdateContent?.({ ...node.data, characters, scenes, props, hasResult: true });
  }
}, []);  // 缺少: characters, scenes, props, onUpdateContent
```

### 影响
- 修改数据后可能不同步到父组件
- 开发模式下 React 会警告
- 可能导致数据不一致

### 建议方案
逐个检查并修复依赖数组：

```javascript
// 修复后的版本
useEffect(() => {
  if (characters.length > 0 || scenes.length > 0 || props.length > 0) {
    onUpdateContent?.({ ...node.data, characters, scenes, props, hasResult: true });
  }
}, [characters, scenes, props, onUpdateContent]);
```

**成本：** 中（需要逐个审查）
**风险：** 低

## 4. RichAgentNode 组件过于臃肿

### 问题描述
`RichAgentNode.jsx` 超过 2500 行，包含 7 个不同的节点组件（Writer, Visual, Director, Producer, Technical, VideoGen, VideoEditor），每个都有重复的 `handleMessagesChange` 逻辑。

### 影响
- 代码维护困难
- 重复代码多
- 难以定位 bug

### 建议方案

**重构方向：提取通用 Hook**

```javascript
// src/hooks/useChatConversation.js
export function useChatConversation({ node, onUpdateContent }) {
  const messagesRef = useRef(node.data?.messages || []);

  const handleMessagesChange = useCallback((newMessages) => {
    if (typeof newMessages === 'function') {
      const resolved = newMessages(messagesRef.current);
      messagesRef.current = resolved;
      onUpdateContent?.({ ...node.data, messages: resolved });
    } else {
      messagesRef.current = newMessages;
      onUpdateContent?.({ ...node.data, messages: newMessages });
    }
  }, [node.data, onUpdateContent]);

  return { messagesRef, handleMessagesChange };
}
```

然后每个节点组件可以简化为：

```javascript
const WriterNodeContent = ({ node, isRunning, onUpdateContent, ... }) => {
  const { handleMessagesChange } = useChatConversation({ node, onUpdateContent });

  return (
    // ... 简化的 JSX
  );
};
```

**成本：** 高（需要较大重构）
**风险：** 中（可能引入新 bug）
**优先级：** 低（当前功能正常）

## 5. 后端异常处理不统一

### 问题描述
业务异常散落在各处，使用 RuntimeException 或专用异常，状态码不统一。

**问题示例：**
- `UserService` 中业务异常使用 `RuntimeException`
- 部分返回 400，部分返回 401/409

### 建议方案

**创建统一的业务异常类：**

```java
// exception/BusinessException.java
public class BusinessException extends RuntimeException {
    private final int statusCode;
    private final String errorCode;

    public BusinessException(String message, int statusCode, String errorCode) {
        super(message);
        this.statusCode = statusCode;
        this.errorCode = errorCode;
    }

    // 常用静态方法
    public static BusinessException accountNotFound() {
        return new BusinessException("账号不存在", 404, "ACCOUNT_NOT_FOUND");
    }

    public static BusinessException accountDisabled() {
        return new BusinessException("账号已被禁用", 403, "ACCOUNT_DISABLED");
    }
}
```

**成本：** 中（需要重构 Service 层）
**风险：** 中
**优先级：** 中
