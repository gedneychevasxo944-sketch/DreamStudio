## 1. 设计原则

### 1.1 产品定位
- **产品类型**: AI 创意工具 / 视频创作平台
- **目标用户**: 内容创作者、视频制作者、导演
- **核心体验**: 专业但易用，AI 辅助创作，流程清晰

### 1.2 设计关键词
- **创意感**: 视觉上有创意气息，但不过度花哨
- **专业感**: 工具感强，值得信赖
- **清晰**: 信息层级清晰，操作路径短
- **沉浸**: 创作时减少干扰，聚焦内容

### 1.3 设计风格
- **风格**: 现代简约 + 玻璃拟态点缀
- **图标**: Lucide / Heroicons（线性风格，2px 描边）
- **插图**: 几何抽象风格，渐变点缀

## 6. 动效规范

### 6.1 性能核心原则 ⚡

**必须使用 GPU 加速的属性**：
```
✅ transform: translate(), scale(), rotate(), skew()
✅ opacity
✅ filter: blur() (谨慎使用)
❌ width, height (触发 layout)
❌ top, left, right, bottom (触发 layout)
❌ margin, padding (触发 layout)
```

**动画性能黄金法则**：
1. **只动画 `transform` 和 `opacity`** — 这两个属性不触发 layout/paint
2. **使用 `will-change`** — 告诉浏览器提前优化，但用完即删
3. **避免布局抖动** — 读写 DOM 分离，批量读写
4. **使用 `translate` 替代 `top/left`** — translate 不触发 layout

### 6.2 时长

| 名称 | 时长 | 用途 |
|------|------|------|
| `duration-instant` | 0ms | 状态切换 |
| `duration-fast` | 100-150ms | 微交互（按钮点击、hover） |
| `duration-normal` | 150-200ms | 一般过渡（展开、收起） |
| `duration-slow` | 200-300ms | 复杂动画（浮窗出现） |
| `duration-slower` | 300-400ms | 页面过渡（路由切换） |

### 6.3 缓动函数

| 名称 | 曲线 | 用途 |
|------|------|------|
| `ease-out` | `cubic-bezier(0, 0, 0.2, 1)` | 进入动画（推荐） |
| `ease-in` | `cubic-bezier(0.4, 0, 1, 1)` | 退出动画 |
| `ease-in-out` | `cubic-bezier(0.4, 0, 0.2, 1)` | 状态变化 |
| `ease-spring` | `cubic-bezier(0.175, 0.885, 0.32, 1.275)` | 弹性效果（谨慎用） |

### 6.4 动画性能检查清单

```
实现前检查：
□ 动画属性是否只有 transform/opacity？
□ 是否避免了 width/height/top/left？
□ will-change 是否正确使用（用完即删）？
□ 是否有读写 DOM 分离？
□ 是否使用了 requestAnimationFrame？

实现后测试：
□ 60fps 是否稳定？（Chrome DevTools Performance）
□ 是否有 jank 或掉帧？
□ 低性能设备上是否流畅？
□ prefers-reduced-motion 是否生效？
```

### 6.5 动画实现代码模板

#### 浮窗出现/消失
```css
/* 好: GPU 加速，性能最佳 */
.dialog {
  opacity: 0;
  transform: scale(0.95) translateY(8px);
  transition: opacity 200ms ease-out, transform 200ms ease-out;
  will-change: transform, opacity;
}
.dialog.visible {
  opacity: 1;
  transform: scale(1) translateY(0);
}
/* 动画结束后移除 will-change */
.dialog.done {
  will-change: auto;
}
```

#### 按钮点击反馈
```css
/* 好: 100ms，使用 transform */
.button:active {
  transform: scale(0.97);
  transition: transform 100ms ease-out;
}
```

#### 拖动动画（对话助手）
```js
// 好: 使用 transform，性能最佳
element.style.transform = `translate(${x}px, ${y}px)`;
// 不好: element.style.left = `${x}px` — 触发 layout
```

#### 主题切换
```css
/* 好: 只动画 opacity，避免闪烁 */
.theme-switch {
  transition: background-color 200ms ease,
              color 150ms ease,
              border-color 150ms ease;
}
```

### 6.6 动画原则

1. **性能优先**: 始终使用 transform/opacity，不触发 layout
2. **响应优先**: 点击反馈 < 100ms
3. **意义清晰**: 每个动画都有目的，不做装饰性动画
4. **可中断**: 动画可被用户操作中断
5. **尊重偏好**: `prefers-reduced-motion` 时禁用复杂动画
6. **进出有别**: 退出动画时长 = 进入动画时长 × 0.7

## 附录: 深浅主题切换

### 切换按钮位置
- **方案A**: 浮窗关闭状态时在导航栏右侧
- **方案B**: 浮窗展开状态时在浮窗标题栏

### 切换动画
- 背景色: 200ms ease-in-out 渐变
- 文字色: 150ms 快速切换
- 避免闪烁: 使用 CSS 变量 + `prefers-color-scheme`

### 自动切换
- 跟随系统: `prefers-color-scheme`
- 手动覆盖: localStorage 存储用户偏好
